Productivity = Active Minutes ÷ 390 (standard daily working minutes) × 100%. Speed % = Target Minutes ÷ Rendered Minutes × 100, capped at 150%. Accuracy is volume-weighted across a person's graded roles for the day (Picker graded downstream by Staging Checker discrepancies; Staging/Outbound Checker graded on their own logged discrepancies; Puller is untracked — shown as "—"). Every personnel on file always holds a rank on whichever board matches their role(s), even with zero logged activity for the selected Activity Day — ranking within a board is by a single selected metric only, with zero-activity personnel always placed last. A person holding roles in both pools (uncommon) appears on both boards, scored only on that board's roles.

Hit = Actual Rendered Time ≤ Target Time for that shipment group; Miss otherwise. Speed % = Target Minutes ÷ Rendered Minutes × 100, averaged across every shipment-group task for that person and capped at 150% (e.g. one task at 100% and a second at 80% averages to 90%) — 100% = exactly on target, above 100% = faster than target. A person holding both roles in a group (e.g. Picker + Puller) has tasks from both merged into one row. Picker Accuracy is graded downstream from Staging Checker discrepancies on the same shipment; Staging/Outbound Checker Accuracy is graded from their own logged discrepancies; Puller Accuracy is not separately tracked (—). Every personnel on file who holds a role in a board's pool always appears on that board, even with zero logged activity for the selected Activity Day — zero-activity personnel show dashes and always sink to the bottom regardless of sort.

✍️ Enter STO / Manual Dispatch

For a rush/off-manifest SO that didn't come in on the daily upload. Every field below is required — this row needs to carry the same complete manifest data as an uploaded one, since Workbench targets, speed, and bracket calculations all depend on it. Select Truck Type first — choosing STO auto-generates a unique Shipment No. (date+time) and SO Number (STO+date+time), and auto-fills OBD/Billing, Ship To Party, SO Amount, and Status for you.

👤 Personnel Roster

Header row required: Personnel, Main Role (Secondary Role / Third Role / Fourth Role columns are optional). Each role value must exactly match: Picker, Staging Checker, Puller, Outbound Checker. Legacy "Designation" header is still accepted as Main Role.

Main / Secondary / Third / Fourth Role define every role a person may be assigned to (there are only 4 possible roles, so this covers every combination). Each role still gets its own activity logs and target — the Role Leaderboards group Picker + Puller into one "Picking Operations" board and Staging Checker + Outbound Checker into one "Checking Operations" board, so a person's work under any of their roles counts on the right board.

📍 Staging Area Masterlist

Header row required: Staging Area — one area name per line after that.

This is the masterlist a Staging Checker picks from when confirming a check — at least one area is required per completion.
